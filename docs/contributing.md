# Contributing to CavrixOS

We welcome contributions! Please follow these guidelines:

1. **Fork the repository** and create a feature branch.
2. **Adhere to Arch Linux standards**: Use `PKGBUILD` for packaging, and follow the Arch package guidelines.
3. **Test your changes**: Build the ISO locally (`sudo make all`) and test it in a VM (`make test`) before submitting a PR.
4. **Keep it minimal**: We aim for a clean, bloat-free system. Justify new packages or services.
5. **Submit a Pull Request**: Provide a clear description of the changes and why they are needed.
